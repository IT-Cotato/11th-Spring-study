### 💡 방향, 다중성

- 방향 : 단방향, 양방향
- 다중성 : 다대일(N:1), 일대다(1:N), 일대일(1:1), 다대다(N:M)

### 💡 객체와 테이블 연관관계의 차이

- 객체를 테이블에 맞추어 모델링
  - 참조 대신에 외래 키를 그대로 사용하는 방법
  - 식별자로 다시 조회 → 객체 지향적인 방법 아님
  - 테이블은 외래 키로 조인을 사용해서 연관된 테이블을 찾음

    ```java
    // Member
    @Column(name = "TEAM_ID")
    private Long teamId;
    ```

- 객체
  - 참조를 사용해서 연관된 객체를 찾음
- 객체 지향 모델링을 해야함 (객체 연관관계 사용)
  - 객체의 참조와 테이블의 외래키를 매핑
  - 참조를 사용해서 연관관계 조회 - 객체 그래프 탐색

    ```java
    // Member
    @ManyToOne
    @JoinColumn(name = "TEAM_ID") // 이 엔티티에서 어떤 컬럼을 외래 키로 만들지 설정
    private Team team;
    ```

### 💡 양방향 연관관계와 연관관계의 주인

- 테이블
  - 단방향에서 양방향으로 바뀌어도 테이블은 변화 없음
  - FK 하나 넣으면 양방향으로 조회 가능함
  - 테이블 연관관계 1개로 양방향 표현 가능
- 객체
  - 반대편 객체에 List 등을 넣음으로써 양방향으로 접근 가능
  - 객체 연관관계 2개로 양방향 표현 가능 → 서로 다른 단방향 관계 2개
    - 회원 → 팀 연관관계 1개 (단방향)
    - 팀 → 회원 연관관계 1개 (단방향)

    ```java
    // Member
    @ManyToOne
    @JoinColumn(name = "TEAM_ID")
    private Team team;
    
    // Team
    @OneToMany(mappedBy = "team") // 연관관계의 주인이 Member 엔티티의 team 필드, 읽기 전용
    List<Member> members = new ArrayList<Member>(); // 초기화를 바로 해주는 것이 관례, null 값 처리 안 해도 됨
    ```

    ```java
    findTeam.getMembers().size() // 이런식으로 역방향 조회 가능
    ```

### 💡 연관관계의 주인(Owner)

- 양방향 매핑 규칙
  - 객체의 두 관계 중 하나를 연관관계 주인으로 지정
  - 연관관계의 주인만이 외래 키를 관리(등록, 수정)
  - 주인이 아닌 쪽은 읽기만 가능
  - 주인은 mappedBy 속성 사용 못 함
  - 주인이 아니면 mappedBy 속성으로 주인 지정
  - 외래 키가 있는 곳을 주인으로 정하면 됨 → DB의 N 쪽이 연관관계의 주인
- 주의할 점
  - 양방향 매핑 시 연관관계의 주인에 값을 입력해야 함
  - 객체 지향적으로 생각하면 항상 양쪽 다 값을 입력해야 함 → 연관관계 편의 메소드 생성

      ```java
      // Member
      public void setTeam(Team team) {
          this.team = team;
          team.getMembers().add(this);
      }
      ```

  - 연관관계 편의 메소드는 한쪽에만 만들기
  - 양방향 매핑 시에 무한 루프 주의

### 💡 양방향 매핑 정리

- 단방향 매핑만으로도 연관관계 매핑은 완료
  - 양방향 경우, 연관관계 편의 메서드나 신경 쓸 것이 많음
  - 그러나, 역방향으로 탐색할 일이 많음
  - 따라서, 일단 단방향 매핑으로 잘 해놓고 나중에 필요할 때 양방향 추가해도 됨
- 외래 키의 위치를 기준으로 연관관계 주인 정하기

### 💡 새롭게 알게 된 점 및 중요한 점 정리

1. 테이블은 외래 키로 조인을 사용해서 연관된 테이블을 찾고, 객체는 참조를 사용해서 연관관계 조회
2. 외래 키가 있는 곳이 연관관계 주인
3. 양방향일 경우 연관관계 편의 메소드 생성해야함
4. 단방향 제대로 해 놓은 다음 필요할 때 양방향 세팅

- 스코프 : 빈이 존재할 수 있는 범위

## 스프링이 지원하는 다양한 스코프 종류

- 싱글톤(Singleton) 스코프
    - 스프링 빈이 기본 스코프
    - 스프링 컨테이너가 시작될 때 딱 하나만 만들어져서 컨테이너가 종료될 때까지 계속 유지되는 가장 넓은 범위의 스코프
    - 요청 시 스프링 컨테이너가 항상 같은 인스턴스 반환
    - 스프링 컨테이너가 시작부터 끝까지 책임짐
    
    ![image.png](attachment:9ef6f1e5-8ab3-4830-a56f-1f81abaa366b:image.png)
    
- 프로토타입(Prototype) 스코프
    - 요청할 때마다 새로운 빈이 생성되는 짧은 범위의 스코프
    - 요청할 때마다 스프링 컨테이너가 새로운 인스턴스 생성 및 반환
    
    ![image.png](attachment:72e78a9e-bc3b-497f-9ae5-537bfec7e48a:image.png)
    
    - 스프링 컨테이너는 생성, 의존관계 주입, 초기화까지만 처리
        - 클라이언트가 직접 관리하고 종료해야 함
        
        ![image.png](attachment:8c9baebf-775f-4f0e-af32-a31582c296cf:image.png)
        
- 웹 관련 스코프
    - 웹 환경에서 사용되는 스코프
    - request, session, application, websocket 등…

## 싱글톤 빈이 프로토타입 빈을 사용할 때 발생하는 문제점

- 싱글톤 빈은 스프링 컨테이너가 생성될 때 딱 한 번 의존관계 주입을 받음
- 이때 프로토타입 빈도 주입을 받게 됨
    - 프로토타입 빈은 주입 시점에 생성된 그 인스턴스로 고정됨
    
    ⇒ 싱글톤 빈이 계속 같은 프로토타입 빈 인스턴스를 사용함(프로토타입의 특징을 제대로 발휘 못 함)
    
- 해결 방법
    - Provider
        - 싱글톤 빈이 프로토타입 빈이 필요할 때마다 스프링 컨테이너에 직접 요청
        - objectProvider(스프링 제공), JSR-330 Provider(자바 표준) 사용
            - 싱글톤 빈 안에서 getObject( ) 또는 get( ) 메서드를 호출할 때마다 스프링 컨테이너를 통해 새로운 프로토타입 빈 인스턴스를 받을 수 있음

## 웹 스코프의 종류와 특징

- 싱글톤처럼 스프링이 종료 시점까지 관리해주지만 프로토타입처럼 짧은 범위의 생명주기를 가짐
- 주요 웹 스코프
    - request : http 요청 하나가 들어오고 나갈 때까지 유지됨. 요청마다 별도의 빈 인스턴스 생성
    - session : HTTP 세션이 생성되고 종료될 때까지 유지됨
    - application : 서블릿 컨텍스트와 같은 범위로 유지됨
    - websocket : 웹 소켓과 동일한 생명주기를 가짐

## 싱글톤 빈에서 request 스코프 빈을 주입 받으려고 하면 오류 발생

- 스프링 애플리케이션이 시작될 때 싱글톤 빈은 생성됨
- request 스코프 빈은 HTTP 요청이 실제로 들어와야 생성되기 때문에 생성 되지 않음
- 해결 방법
    - Provider 사용
        - 싱글톤 빈에서 Provider를 주입 받고 실제 request 스코프 빈이 필요할 때 Provider의 getObject( ) 메서드를 호출해 빈을 가져옴
        - 필요할 때까지 빈 생성을 미루는 지연 로딩 느낌
    - 프록시(Proxy) 사용
        - request 스코프 빈에 @Scope 애노테이션을 설정할 때 proxyMode = ScopedProxyMode.TARGET_CLASS 옵션 추가
            - 스프링이 request 스코프 빈의 프록시 객체를 만들어 싱글톤 빈에 미리 주입해줌
            - 이 프록시 객체는 실제 request 스코프 빈처럼 활동하지만 실제로는 요청이 올 때까지 내부에서 진짜 빈을 생성하고 그 메서드를 대신 호출해주는 역할을 함
        - 필요할 때까지 빈 생성을 미루는 지연 로딩

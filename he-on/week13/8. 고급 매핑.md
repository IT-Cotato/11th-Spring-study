## 1. 상속관계 매핑 개념

- **상속관계 매핑 :** 객체지향 프로그래밍에서의 상속 구조를 데이터베이스 테이블에 맞게 변환하는 방법
- 데이터베이스는 객체의 상속 구조와 달리, 슈퍼타입-서브타입 관계를 명확하게 지원하지 않음
    
    → 매핑하는 다양한 전략이 필요
    
- 상속관계 매핑의 주요 전략 : 조인 전략, 단일 테이블 전략, 구현 클래스마다 테이블 전략
- 각 전략은 @Inheritance 어노테이션의 strategy 속성으로 지정
    - @DiscriminatorColumn, @DiscriminatorValue 어노테이션을 함께 사용

## 2. 조인 전략

- **조인 전략(JOINED) :** 슈퍼타입과 서브타입을 각각 별도의 테이블로 만들고, 조회 시 조인을 통해 데이터를 가져오는 방식
    - 장점:
        - 테이블 정규화가 잘 됨
        - 외래 키 참조 무결성 제약조건을 활용할 수 있음
        - 저장공간이 효율적
    - 단점:
        - 조회 시 조인을 많이 사용하므로 성능이 저하될 수 있음
        - 조회 쿼리가 복잡해짐
        - 데이터 저장 시 INSERT SQL이 2번 호출됨

## 3. 단일 테이블 전략

- **단일 테이블 전략(SINGLE_TABLE) :** 모든 상속 구조의 데이터를 하나의 테이블에 저장하는 방식
    - 장점:
        - 조인이 필요 없으므로 일반적으로 조회 성능이 빠름
        - 구조가 단순
    - 단점:
        - 자식 엔티티가 단일 테이블에 모두 저장되므로, 매핑한 컬럼은 모두 null을 허용해야 함
        - 모든 데이터를 저장하므로 테이블이 커질 수 있음
        - 상황에 따라 조회 성능이 오히려 느려질 수 있음

## 4. 구현 클래스마다 테이블 전략

- **구현 클래스마다 테이블 전략(TABLE_PER_CLASS) :** 각 자식 클래스마다 별도의 테이블을 생성하는 방식
    - 장점:
        - not null 제약조건 사용 가능
        - 서브타입을 명확하게 구분해서 처리할 때 효과적
    - 단점:
        - 자식 테이블을 통합해서 쿼리하기 어려움
        - 여러 자식 테이블을 함께 조회할 때 성능이 느림(UNION SQL 필요)
        - 데이터베이스 설계자와 ORM 전문가 모두 추천하지 않는 전략임

## 5. @MappedSuperclass 활용

- **@MappedSuperclass :**  공통 매핑 정보가 필요할 때 사용하는 어노테이션
    - 주로 id, name, 등록일, 수정일, 등록자, 수정자 등 엔티티가 공통으로 가지는 정보를 모을 때 사용
    - @MappedSuperclass로 지정된 클래스는 테이블과 매핑되지 않으며, 엔티티가 아님
    - 부모 클래스의 매핑 정보만 자식 클래스에 제공하며, 직접 조회나 검색이 불가함
    - 추상 클래스로 만드는 것이 권장됨
    - @Entity 클래스만 테이블과 매핑되고, @MappedSuperclass는 엔티티나 다른 @MappedSuperclass에서 상속 가능

## 6. 실전 예제 및 요구사항

- ex: 추가 상품의 종류로 음반, 도서, 영화가 있으며, 이후 더 확장될 수 있음
    - 모든 데이터는 등록일과 수정일이 필수
- 도메인 모델 설계 시, 상속관계 매핑과 @MappedSuperclass를 활용하여 공통 속성을 관리

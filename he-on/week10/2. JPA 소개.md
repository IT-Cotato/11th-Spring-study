## 1. JPA 개요

- **JPA**는 Java Persistence API의 약자, 자바 진영의 ORM(객체 관계 매핑) 기술 표준
- JPA는 애플리케이션과 JDBC 사이에서 동작, 객체와 관계형 데이터베이스 간의 매핑을 처리한다.

## 2. SQL 중심 개발의 문제점

- SQL 중심의 개발은 무한 반복과 지루한 코드 작성을 초래
    - ex: CRUD(Create, Read, Update, Delete) 작업을 SQL로 직접 작성해야 함
- 객체 지향 프로그래밍의 특성과 관계형 데이터베이스의 불일치로 인해 개발이 복잡해짐

## 3. JPA의 필요성

- JPA는 SQL 중심의 개발에서 객체 중심으로의 전환을 가능하게 하며, 패러다임의 불일치를 해결
- JPA는 데이터 접근의 추상화와 벤더 독립성을 제공하여 생산성과 유지보수성을 높임

## 4. JPA의 주요 기능

- JPA를 사용하면 CRUD 작업을 간편하게 수행할 수 있음
- ex : 저장 - `jpa.persist(member)`, 조회 -  `Member member = jpa.find(memberId)`

## 5. JPA와 상속

- JPA는 객체 상속 관계를 지원, 개발자는 저장과 조회 작업을 간편하게 처리할 수 있음.
- ex :  `jpa.persist(album)`로 Album 객체를 저장하고, `Album album = jpa.find(Album.class, albumId)`로 조회할 수 있음

## 6. JPA의 성능 최적화 기능

- JPA는 1차 캐시와 동일성을 보장하여 같은 트랜잭션 내에서 동일한 엔티티를 반환
- 또한, 트랜잭션을 지원하는 쓰기 지연 기능을 통해 INSERT, UPDATE, DELETE SQL을 커밋 시점에 모아서 처리할 수 있음

## 7. 지연 로딩과 즉시 로딩

- JPA는 지연 로딩과 즉시 로딩을 지원하여 객체가 실제 사용될 때 로딩하거나, JOIN SQL로 연관된 객체를 미리 조회할 수 있음

## 8. ORM의 중요성

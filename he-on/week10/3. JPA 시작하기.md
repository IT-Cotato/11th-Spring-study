## 1. 데이터베이스 방언 및 JPA의 관계

- JPA는 특정 **데이터베이스에 종속적이지** 않으며, 각 데이터베이스마다 제공하는 SQL 문법과 함수는 다를 수 있음
- 데이터베이스 함수를 통해 **문자열을 자르는 방식**이 다르며, SQL 표준은 SUBSTRING()을 사용하고, Oracle은 이를 다른 방식으로 처리
- **방언**은 SQL 표준을 지키지 않는 특정 데이터베이스만의 고유한 기능 의미.
- 하이버네이트는 다양한 데이터베이스 방언을 지원, 각 데이터베이스에 맞는 **다이얼렉트를 지정해야** 함
- ex :
    - Oracle 10g ⇒ `org.hibernate.dialect.Oracle10gDialect`
    - MySQL ⇒ `org.hibernate.dialect.MySQL5InnoDBDialect`

## 2. JPA 구동 방식 개요

- **방언 지원**은 JPA의 중요한 기능 중 하나
- JPA를 통해 애플리케이션 개발을 위한 동작 방식 이해 가능

## 3. JPA 객체 및 테이블 생성과 매핑

- **@Entity** 어노테이션은 JPA가 관리할 객체를 정의
    - 이로 인해 객체와 데이터베이스 테이블 간의 매핑이 이루어짐
- **@Id** 어노테이션은 데이터베이스의 **PK**(primary key)와 매핑되는 필드를 나타냄
    - 보통 클래스 내에서 사용
- ex : `Member` 클래스는 id와 name 필드를 가지며, 여기서 id는 **Long** 타입으로 지정
- 테이블 생성 쿼리는 `create table Member` 문을 통해 실행되며, 이에 따라 회원 테이블이 정의됨
- JPA의 모든 데이터 변경 작업은 반드시 **트랜잭션** 내에서 실행되어야 하며, 이를 통해 데이터의 일관성 유지 가능

## 4. JPQL 소개 및 개요

- **JPQL**은 JPA에서 사용하는 **객체 지향 쿼리 언어**로, SQL을 추상화하여 사용
- JPQL은 **SELECT, FROM, WHERE, GROUP BY, HAVING, JOIN**과 같은 SQL 문법을 지원하지만, 데이터베이스의 테이블이 아닌 **엔티티 객체**를 대상으로 쿼리
- JPQL을 사용하면 특정 **데이터베이스 SQL에 의존하지 않으면서** 객체 지향적으로 데이터를 검색할 수 있음
- JPQL은 애플리케이션이 필요한 데이터만을 **DB에서 불러오기 위한 쿼리**를 작성하는 데 사용되며, SQL로 변환되는 과정을 포함
- ex : 특정 조건에 맞는 회원을 검색하고 싶을 때 JPQL을 활용하여 간단하게 조회할 수 있다.

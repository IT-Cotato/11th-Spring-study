## 1. 연관관계 매핑 개요

- **연관관계 매핑 :** 데이터베이스 테이블 간의 관계를 객체 모델에 맞게 매핑하는 것
- 매핑 시 고려해야 할 **3가지 주요 사항**
    - 연관관계의 **다중성**: 다대일(N:1), 일대다(1:N), 일대일(1:1), 다대다(N:M)
    - **단방향**과 **양방향** 매핑: 한쪽만 참조하거나, 양쪽이 서로 참조하는 구조
    - **연관관계의 주인**: 외래 키를 관리하는 쪽을 지정해야 하며, 주인이 아닌 쪽은 단순 조회만 가능
- ex :
    - 객체 양방향 관계: A→B, B→A처럼 두 군데에서 참조
    - 테이블은 외래 키 하나로 두 테이블이 연관
    

## 2. 다대일(N:1) 매핑

- **다대일 관계**는 가장 많이 사용되는 연관관계
- **단방향 다대일**: 한쪽(다수)이 다른 한쪽(일)을 참조
- **양방향 다대일**: 양쪽 모두 서로를 참조할 수 있음
    - 외래 키가 있는 쪽이 참조하도록 설계
- **연관관계의 주인**은 외래 키를 가진 쪽

## 3. 일대다(1:N) 매핑

- **일대다 관계**는 테이블 구조와 객체 구조가 달라서 주의가 필요
    - 항상 다(N) 쪽에 외래 키가 있음
    - 단방향 매핑 시 @JoinColumn을 꼭 사용해야 하며, 그렇지 않으면 조인 테이블이 추가됨
- **단방향 일대다**의 단점:
    - 외래 키가 다른 테이블에 있어 관리 복잡
    - 추가로 UPDATE SQL이 실행됨
    - **다대일 양방향 매핑**을 사용하는 것이 더 효율적임
- **양방향 일대다**:
    - 읽기 전용 필드를 사용하여 다대일 양방향을 구현함
    - @JoinColumn(insertable=false, updatable=false)로 읽기 전용 필드 지정 가능
    

## 4. 일대일(1:1) 매핑

- **일대일 관계**는 외래 키를 어느 테이블에 둘지 선택 가능
    - 외래 키에 유니크(UNI) 제약조건을 추가해야 함
- **주 테이블에 외래 키**:
    - 단방향: 주 테이블이 대상 테이블을 참조
    - 양방향: 주 테이블이 외래 키를 가지고, 반대편은 mappedBy로 매핑
    - 외래 키가 있는 곳이 연관관계의 주인
- **대상 테이블에 외래 키**:
    - 단방향: JPA에서 지원하지 않음
    - 양방향: 주 테이블과 대상 테이블 모두 외래 키를 가질 수 있음
- **정리**:
    - 주 테이블에 외래 키를 두는 방식은 객체지향 개발자와 JPA 매핑에 유리함
        - 장점: 주 테이블만 조회해도 대상 테이블 데이터 확인 가능
        - 단점: 값이 없으면 외래 키에 null 허용
    - 대상 테이블에 외래 키를 두는 방식은 전통적인 데이터베이스 개발자에게 선호됨
        - 장점: 일대일에서 일대다로 변경 시 테이블 구조 유지
        - 단점: 프록시 기능 한계로 항상 즉시 로딩됨
        

## 5. 다대다(N:M) 매핑

- **관계형 데이터베이스**는 다대다 관계를 직접 표현할 수 없으므로, **연결 테이블**을 추가해 일대다, 다대일 관계로 풀어내야 함
- **객체 모델**에서는 컬렉션을 사용해 다대다 관계를 표현할 수 있음
- **@ManyToMany, @JoinTable**을 사용해 매핑 가능하며, 단방향/양방향 지정 가능
- **실무 한계**:
    - 연결 테이블이 단순 연결만 하는 것이 아니라 추가 데이터(예: 주문시간, 수량 등)가 들어올 수 있음
    - 실무에서는 @ManyToMany를 거의 사용하지 않음
- **해결 방법**:
    - 연결 테이블을 엔티티로 승격하여 @OneToMany, @ManyToOne으로 매핑
    

## 6. 실전 예제 및 추가 설명

- **실전 예제**:
    - 배송과 주문: 1:1(@OneToOne) 관계
    - 상품과 카테고리: N:M(@ManyToMany) 관계
- **N:M 관계의 실제 구현**:
    - 테이블은 중간 테이블을 이용해 1:N, N:1로 풀어냄
    - 실전에서는 중간 테이블이 단순하지 않아 @ManyToMany 사용에 제약이 있음(필드 추가 불가 등)
- **@JoinColumn 속성**:
    - 외래 키 이름, 참조 대상 컬럼명, 제약조건 등 다양한 속성 지정 가능
    - 기본값: 필드명 + _ + 참조 테이블의 기본 키 컬럼명
    

## 7. 주요 어노테이션 속성 정리

- **@ManyToOne** (다대일 관계)
    - 주요 속성: optional, fetch, cascade, targetEntity
        - optional: false로 설정하면 연관 엔티티가 항상 존재해야 함
        - fetch: 글로벌 페치 전략 설정
        - cascade: 영속성 전이 기능
        - targetEntity: 연관 엔티티 타입 정보(거의 사용하지 않음)
    - 기본값: FetchType.EAGER
- **@OneToMany** (일대다 관계)
    - 주요 속성: mappedBy, fetch, cascade, targetEntity
        - mappedBy: 연관관계의 주인 필드 선택
        - fetch, cascade, targetEntity: 위와 동일
    - 기본값: FetchType.LAZY

## 애플리케이션 아키택처

### 계층형 구조

- 웹 계층 → 사용자 요청 받기
    - controller
    - web
- 서비스 계층
    - 비즈니스 로직, 트랜잭션 처리
- 리포지토리 계층
    - JPA 직접 사용 - 엔티티 매니저와 상호작용
- 도메인 계층
    - 엔티티

*기능이 많아지면 파일이 섞여서 찾기 힘듦

## 개발 순서

→ 핵심 비즈니스 로직과 데이터 접근 부분을 탄탄하게 만들고 나서 사용자 인터페이스를 붙이는 방식

1. 서비스, 리포지토리 계층 개발
2. 테스트 케이스 작성 후 검증
3. 웹 계층 적용

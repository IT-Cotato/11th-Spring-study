## 1. 영속성의 내부 구조와 중요성

- JPA에서 '영속성'은 엔티티를 데이터베이스에 영구 저장하는 환경을 의미
- 영속성 관리는 JPA의 핵심 개념으로, 엔티티를 어떻게 저장하고 관리하는지 이해하는 것이 중요

## 2. 영속성 컨텍스트의 개념과 역할

- **영속성 컨텍스트는 엔티티를 영구 저장하는 환경**으로, 엔티티 매니저(EntityManager)가 관리
- 엔티티 매니저와 영속성 컨텍스트는 JPA에서 1:1 또는 N:1 관계를 가짐
- 영속성 컨텍스트는 눈에 보이지 않는 논리적 개념으로, J2SE 환경에서는 엔티티 매니저와 1:1 관계, J2EE 또는 스프링 환경에서는 N:1 관계
- 엔티티 매니저는 엔티티를 관리하는 객체이며, 영속성 컨텍스트는 엔티티의 생명주기를 담당

## 3. 엔티티의 생명주기와 상태

- 엔티티는 네 가지 상태를 가짐:
    - 비영속 (new/transient): 새로 생성된 상태, 영속성 컨텍스트와 관계 없음
    - 영속 (managed): 영속성 컨텍스트에 저장되어 관리됨
    - 준영속 (detached): 영속성 컨텍스트에서 분리된 상태
    - 삭제 (removed): 삭제된 상태
- 엔티티의 생명주기 예시:
    - 비영속: `Member member = new Member();`
    - 영속: `em.persist(member);`
    - 준영속: `em.detach(member);`
    - 삭제: `em.remove(member);`
- 영속성 컨텍스트의 이점:
    - 1차 캐시를 통해 성능 향상
    - 동일성 보장: 같은 엔티티는 항상 같은 인스턴스
    - 트랜잭션 지원, 쓰기 지연, 변경 감지, 지연 로딩 가능

## 4. 엔티티 조회와 1차 캐시

- 엔티티 조회 방법:
    - 새 엔티티 생성 후 `em.persist()`로 1차 캐시에 저장
    - `em.find()`로 데이터베이스 또는 1차 캐시에서 조회 가능
- 1차 캐시의 특징:
    - 같은 엔티티를 여러 번 조회해도 데이터베이스에 다시 요청하지 않음
    - 동일성 보장: `a == b`가 `true`
- 데이터베이스에서 조회하는 경우:
    - `em.find()`로 엔티티를 찾음
- 1차 캐시로 인해 반복 가능한 읽기(REPEATABLE READ) 등급의 트랜잭션 격리 수준이 애플리케이션 차원에서 제공됨

## 5. 엔티티 등록과 트랜잭션

- 엔티티 등록 과정:
    - `em.persist()` 호출 시 INSERT SQL은 트랜잭션 커밋 시점에 데이터베이스로 전송됨
    - 트랜잭션 시작 후 `persist()` 호출, 커밋 시점에 실제 저장됨
- 트랜잭션 예시:
    - `EntityTransaction transaction = em.getTransaction();`
    - `transaction.begin();`
    - 엔티티 등록 후 `transaction.commit();`
- 데이터 변경 시:
    - 엔티티를 조회 후 값 변경.
    - 변경 감지(Dirty Checking)가 활성화되어, 커밋 시점에 변경 내용이 데이터베이스에 반영됨

## 6. 엔티티 수정과 변경 감지

- 엔티티 수정 방법:
    - 영속 상태의 엔티티를 조회 후 필드 값을 변경하면 됨
    - 별도의 `update()` 호출 필요 없음
- 변경 감지(Dirty Checking):
    - 엔티티의 변경 내용을 감지하여, 커밋 시점에 자동으로 SQL 수정 쿼리 생성
- 플러시(Flush):
    - 영속성 컨텍스트의 변경 내용을 데이터베이스에 반영하는 작업
    - `em.flush()`로 수동 호출 가능
    - 트랜잭션 커밋 시 자동으로 호출됨
    - JPQL 실행 시에도 자동 호출됨
- 플러시 모드:
    - `FlushModeType.AUTO`: 커밋 또는 쿼리 실행 시 플러시
    - `FlushModeType.COMMIT`: 커밋 시에만 플러시

## 7. 플러시와 영속성 컨텍스트 동기화

- 플러시의 역할:
    - 변경된 엔티티를 SQL로 만들어 데이터베이스에 전송
    - 트랜잭션이 끝나기 전에만 수행됨
- 플러시 호출 방법:
    - `em.flush()` 수동 호출
    - 트랜잭션 커밋 또는 JPQL 쿼리 실행 시 자동 호출
- 플러시의 핵심:
    - 영속성 컨텍스트와 데이터베이스를 동기화하는 것임

## 8. 준영속 상태와 전환 방법

- 준영속 상태:
    - 영속성 컨텍스트에서 엔티티가 분리된 상태
    - 이 상태의 엔티티는 영속성 컨텍스트의 기능을 사용할 수 없음
- 준영속으로 전환하는 방법:
    - 특정 엔티티만 분리: `em.detach(entity)`
    - 영속성 컨텍스트 전체 초기화: `em.clear()`
    - 엔티티 매니저 종료: `em.close()`
- 영속성 컨텍스트를 종료하거나 분리하면, 엔티티는 더 이상 관리되지 않음

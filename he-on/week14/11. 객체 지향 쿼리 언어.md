## 1. 객체지향 쿼리 언어 개요

- **객체지향 쿼리 언어(JPQL) : JPA에서 제공하는 쿼리 언어,** SQL을 추상화해 엔티티 객체(Entity)를 대상으로 쿼리
    - JPQL, JPA Criteria, QueryDSL, 네이티브 SQL, JDBC API, MyBatis, SpringJdbcTemplate 등 다양한 쿼리 방법 지원
- **JPQL의 특징**
    - **테이블이 아닌 엔티티 객체를 대상으로 검색**
        - SQL은 테이블을 조회하지만 JPQL은 엔티티를 조회
    - SQL과 유사한 문법(SELECT, FROM, WHERE, GROUP BY, HAVING, JOIN 등)
    - 데이터베이스에 의존하지 않음
    - 실제로는 SQL로 변환되어 실행됨
- **JPQL 예시**
    - `"select m From Member m where m.name like ‘%hello%'";`
    - 실행 시 실제 SQL로 변환되어 동작

## 2. 다양한 쿼리 작성 방법

- **JPQL**: 객체를 대상으로 쿼리 작성, SQL로 변환되어 실행
- **Criteria**: 자바 코드로 JPQL을 작성할 수 있는 빌더 역할, 공식 기능이나 복잡해서 실무에서는 QueryDSL 권장
    - 예시: CriteriaBuilder, CriteriaQuery, Root 사용
- **QueryDSL**: 자바 코드로 JPQL을 작성, 컴파일 시점에 문법 오류 확인 가능, 동적 쿼리 작성에 편리, 실무 사용 권장
    - 예시: JPAQueryFactory, QMember 사용
- **네이티브 SQL**: JPQL로 해결할 수 없는 DB 종속 기능(예: 오라클 CONNECT BY, SQL 힌트 등) 직접 사용
    - 예시: `em.createNativeQuery(sql, Member.class).getResultList();`
- **JDBC 직접 사용, SpringJdbcTemplate, MyBatis 등**: JPA와 함께 사용 가능, 영속성 컨텍스트를 수동 플러시 필요

## 3. JPQL 기본 문법과 기능

- **JPQL은 객체지향 쿼리 언어**로, 엔티티 객체를 대상으로 쿼리
- **JPQL 문법**
    - SELECT, FROM, WHERE, GROUP BY, HAVING, ORDER BY, UPDATE, DELETE 등 SQL과 유사
    - 엔티티와 속성은 대소문자 구분, 키워드는 구분하지 않음
    - 엔티티 이름 사용, 별칭 필수
- **집합 함수 및 정렬**
    - COUNT, SUM, AVG, MAX, MIN 등 집합 함수 지원
    - GROUP BY, HAVING, ORDER BY 지원
- **TypedQuery와 Query**
    - 반환 타입이 명확할 때 TypedQuery, 아닐 때 Query 사용
- **조회 API**
    - `getResultList()`: 결과가 하나 이상일 때, 없으면 빈 리스트 반환
    - `getSingleResult()`: 결과가 정확히 하나, 없으면 예외 발생
- **파라미터 바인딩**
    - 이름 기준: `:username`
    - 위치 기준: `?1`

## 4. 프로젝션과 페이징

- **프로젝션**: SELECT 절에 조회할 대상을 지정하는 것
    - 엔티티, 임베디드 타입, 스칼라 타입(숫자, 문자 등) 가능
    - 여러 값 조회 시 Object[], DTO 생성자 사용 가능
- **페이징 API**
    - `setFirstResult(int startPosition)`: 조회 시작 위치 지정(0부터 시작)
    - `setMaxResults(int maxResult)`: 조회할 데이터 수 지정
    - 예시: `.setFirstResult(10).setMaxResults(20)`
    - DB 방언에 따라 SQL이 다르게 생성됨(MySQL: LIMIT, Oracle: ROWNUM)

## 5. 조인과 경로 표현식

- **조인 종류**
    - 내부 조인(INNER JOIN), 외부 조인(LEFT OUTER JOIN), 세타 조인(연관관계 없는 엔티티 조인)
- **ON 절 활용**
    - 조인 대상 필터링, 연관관계 없는 엔티티 외부 조인 가능
    - 예시: 팀 이름이 'A'인 팀만 조인, 회원 이름과 팀 이름이 같은 대상 외부 조인
- **서브쿼리**
    - WHERE, HAVING 절에서만 사용 가능, FROM 절은 불가
    - 예시: 평균 나이보다 많은 회원, 주문한 적 있는 고객 등
- **경로 표현식**
    - 상태 필드: 값 저장용 필드(ex: m.username)
    - 연관 필드: 단일 값(@ManyToOne, @OneToOne), 컬렉션 값(@OneToMany, @ManyToMany)
    - 상태 필드는 탐색 끝, 단일 값 연관 경로는 묵시적 내부 조인 발생, 컬렉션 값 연관 경로는 묵시적 내부 조인 발생
    - 명시적 조인 사용 권장, 묵시적 조인은 SQL 튜닝에 불리

## 6. 페치 조인(fetch join)

- **페치 조인**: 성능 최적화를 위해 연관된 엔티티나 컬렉션을 SQL 한 번에 함께 조회하는 기능
    - `join fetch` 명령어 사용
    - 예시: 회원과 팀을 한 번에 조회, 팀과 회원을 한 번에 조회
- **DISTINCT와 페치 조인**
    - JPQL의 DISTINCT는 SQL에 DISTINCT 추가 + 애플리케이션에서 엔티티 중복 제거
    - 하이버네이트6부터는 DISTINCT 없이도 중복 제거 자동 적용
- **일반 조인과의 차이**
    - 일반 조인은 연관된 엔티티를 함께 조회하지 않음, 페치 조인은 객체 그래프를 SQL 한 번에 조회
- **페치 조인의 한계**
    - 별칭 사용 불가(하이버네이트는 가능하나 권장하지 않음)
    - 둘 이상의 컬렉션 페치 조인 불가, 컬렉션 페치 조인 시 페이징 불가
    - 단일 값 연관 필드는 페이징 가능
    - 글로벌 로딩 전략보다 페치 조인이 우선 적용됨
    - 테이블 조인 결과가 엔티티 모양과 다르면 DTO로 반환하는 것이 효과적

## 7. 다형성 쿼리

- **TYPE 조회**: 상속 구조에서 특정 자식 타입만 조회 가능
    - 예시: `select i from Item i where type(i) IN (Book, Movie)`
- **TREAT(JPA 2.1)**: 자바의 타입 캐스팅과 유사, 부모 타입을 특정 자식 타입으로 다룰 때 사용
    - 예시: `select i from Item i where treat(i as Book).author = ‘kim’`

## 8. 엔티티 직접 사용

- **JPQL에서 엔티티를 직접 사용할 수 있음**
    - 기본 키 값 대신 엔티티 객체를 직접 비교 가능
    - 파라미터로 엔티티 전달 시 SQL에서는 해당 엔티티의 기본 키 값 사용
    - 외래 키 값도 엔티티로 비교 가능
    - 예시: `select m from Member m where m = :member`

## 9. Named 쿼리

- **Named 쿼리**: 미리 정의한 정적 쿼리에 이름을 부여해 재사용 가능
    - 어노테이션(@NamedQuery) 또는 XML에 정의 가능
    - 애플리케이션 로딩 시점에 쿼리 검증 및 초기화
    - XML 설정이 항상 우선권을 가짐, 운영 환경에 따라 다른 XML 배포 가능

## 10. 벌크 연산

- **벌크 연산**: 쿼리 한 번으로 여러 엔티티의 데이터를 변경(UPDATE, DELETE)
    - 예시: 재고가 10개 미만인 모든 상품의 가격을 10% 상승시키기
    - `executeUpdate()`의 결과는 영향받은 엔티티 수 반환
- **주의사항**
    - 벌크 연산은 영속성 컨텍스트를 무시하고 데이터베이스에 직접 반영됨
    - 벌크 연산 후에는 영속성 컨텍스트를 초기화해야 함

QueryDSL = 자바 코드로 JPQL을 타입 안정성 있게 작성하는 라이브러리

→ JPQL을 자바 코드로 더 안전하고 편하게 쓰는 도구

Fetch Join = 연관된 객체를 한 번의 SQL로 같이 가져오는 기능 (N+1 문제 해결)

→  JPA가 자동으로 연관 객체를 안 불러오는 문제(N+1)를 해결하는 기능

## View 환경 설정

thymeleaf - 스프링 부트에서 공식적으로 지원하는 템플릿 엔진

- 정의
    - HTML 안에 데이터를 넣을 수 있게 도와주는 템플릿 엔진
    - 스프링 부트는 타임리프 기본 지원
- 기본 흐름
    - resources/templates 폴더에 .html 파일 생성
    - 컨트롤러에서 해당 “파일 이름”을 문자열로 반환
    - 타임리프가 그 이름의 .html 파일을 자동으로 찾아서 보여줌
- HTML 파일엔 다음 속성 추가 필요⇒ `xmlns:th="http://www.thymeleaf.org"`
- `th:text` 같은 속성을 사용해 데이터 출력
    - ex : `span th:text=”${name}”></span`
        - name 이라는 값을 화면에 출력해줌
- 컨트롤러에서 뷰 이름 반환 시 자동으로 매핑됨
    - 컨트롤러에서 리턴 : return “hello”;
    - 스프링이 자동으로 경로랑 확장자를 붙여줌(매핑함)
        - “hello” → “classpath:/templates/hello.html”
- HTML 파일 수정 시 확인 방법
    - spring-boot-devtools 라이브러리 추가
    - 인텔리제이에서 수동으로  build - recompile 눌러서 반영

## JPA와 데이터베이스 설정

application.yml 파일

- 데이터 소스 설정
    - H2 DB의 URL
    - 사용자 이름
    - 비밀번호
    - 드라이버 클래스 이름 등
- JPA 설정
    - spring.jpa.hibernate.ddl-auto : create
        - 애플리케이션 실행 시점에 테이블 drop 후 다시 생성
        - ⇒ 애플리케이션을 껐다 켜면 테이블에 저장된 데이터는 전부 날아감
        - 개발 초기에 자주 사용
    - 모든 로그 출력은 가급적 로거를 통해 남김
        - show_sql : 옵션은 System.out에 하이버네이트 실행 SQL을 남김
        - org.hibernate.SQL : 옵션은 로거를 통해 하이버네이트 실행 SQL을 남김
- 주의사항
    - 띄어쓰기로 계층 구분
    - ⇒ 띄어쓰기 2칸 필수

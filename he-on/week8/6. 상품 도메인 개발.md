## 엔티티

- 데이터와 관련된 비즈니스 로직 포함(응집도 높임)
    
    ⇒ 데이터 자체에 그 데이터를 가지고 뭘 할 수 있는지, 어떤 규칙이 적용되는지를 함께 정의하는 것
    
    ⇒ 상품 엔티티 - 재고를 늘리거나(addStock) 줄이는 기능(removeStock)
    

```jsx
@Inheritance(strategy = InheritanceType.SINGLE_TABLE)
@DiscriminatorColumn(name = "dtype")
```

@Inheritance(strategy = …) : 상속된 엔티티들을 테이블에 어떻게 저장할 지 결정

- InheritanceType.SINGLE_TABLE : 부모 클래스와 그 자식 클래스들을 하나의 테이블에 모두 저장
    - 필요 없는 필드는 NULL 처리
    - 하나의 테이블이 다양한 타입을 모두 수용하므로 타입 식별을 위해 dtype 컬럼 사용
- InheritanceType.JOINED : 부모와 자식을 각각 테이블에 저장. 조회 시 조인
    - 조인 비용이 큼 → 성능 저하
- InheritanceType.TABLE_PER_CLASS : 자식마다 온전히 독립된 테이블 생성(부모 테이블 없음)
    - 비효율

@DiscriminatiorColumn : 저장된 데이터가 어떤 자식 클래스에 해당하는지 구분

- dtype : 레코드가 어떤 자식 클래스에서 왔는지 식별

## 리포지토리

- 데이터베이스와 상호작용하는 역할
    - 데이터 저장(save)
        - id가 없으면 persist()
        - id가 있으면 수정으로 보고 merge()
    - 탐색(findOne)
    - 목록 가져오기(findAll)

## 서비스

- 리포지토리를 활용해 비즈니스 로직 수행
    - 리포지토리를 이용해 데이터 가져오기, 저장하기
    - 필요한 비즈니스 로직 수행

# 섹션 6. 싱글톤 컨테이너

### 싱글톤 패턴이란?
싱글톤 패턴은 객체가 JVM 내에 단 하나만 존재하도록 보장하는 디자인 패턴  
여러 클라이언트가 같은 인스턴스를 공유하게 해서 메모리 낭비를 줄여줌  

### 기존 싱글톤 패턴의 문제점
```java
public class SingletonService {
    // 1. static 영역에 객체를 딱 1개만 생성
    private static final SingletonService instance = new SingletonService();

    // 2. public static 메서드를 통해서만 조회 가능
    public static SingletonService getInstance() {
        return instance;
    }

    // 3. private 생성자로 외부에서 new 키워드 사용 방지
    private SingletonService() {
    }

    public void logic() {
        System.out.println("싱글톤 객체 로직 호출");
    }
}
```
- 구현 코드 복잡함
- private 생성자로 인해 상속이 어려움
- 유연성 떨어짐
- 테스트 어려움  

### 스프링 컨테이너와 싱글톤
스프링 컨테이너는 기존 싱글톤 패턴의 단점을 해결하면서 빈을 싱글톤으로 관리함  

- 개발자가 직접 싱글톤 패턴을 구현할 필요 X
- 스프링이 알아서 싱글톤으로 객체를 관리
- DIP, OCP 원칙을 지키면서 싱글톤 구현 가능

### @Configuration과 싱글톤

```java
@Configuration
public class AppConfig {
	@Bean
	public MemberService memberService() {
		return new MemberServiceImpl(memberRepository());
	}

	@Bean
	public OrderService orderService() {
		return new OrderServiceImpl(memberRepository(), discountPolicy());
	}

	@Bean
	public MemberRepository memberRepository() {
		return new MemoryMemberRepository();
	}
	// ...
}
```
- 위 코드에서 memberRepository()가 여러 번 호출되는 것처럼 보이지만, 실제로는 한 번만 생성됨

#### 어떻게?

- 스프링은 @Configuration이 붙은 클래스를 CGLIB라는 바이트코드 조작 라이브러리를 사용해 
해당 클래스를 상속한 프록시 객체를 생성함 (AppConfig -> AppConfig@CGLIB)
- 이 프록시는 @Bean 메서드 호출을 가로채서 특별한 로직을 추가함
  - 스프링 컨테이너에 해당 빈이 이미 존재하는지 확인
  - 이미 존재하면 컨테이너에서 찾아서 반환하고, 없으면 생성 후 컨테이너에 등록

#### @Configuration 없이 @Bean만 붙이면?

- CGLIB 프록시가 생성되지 않아 메서드 호출 가로채기가 없음
- memberRepository()가 호출될 때마다 새로운 MemoryMemberRepository 인스턴스가 생성됨
- 스프링 컨테이너에는 하나의 MemberRepository 빈이 등록되지만, MemberService와 OrderService는 각각 다른 MemoryMemberRepository 인스턴스를 참조하게 됨

**대안**  
@Autowired를 사용해 이미 등록된 빈을 주입받기
```java
public class AppConfig {
    @Autowired MemberRepository memberRepository;
    
    @Bean
    public MemberService memberService() {
        return new MemberServiceImpl(memberRepository);
    }
    // ...
}
```
- 하지만 이 방법보다는 @Configuration을 사용하는 것이 스프링의 의도된 사용법이고 더 명확함

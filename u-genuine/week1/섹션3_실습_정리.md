# 섹션3. 스프링 핵심 원리 이해1 - 예제 만들기

### 인터페이스와 구현체 분리의 중요성
```java
public class MemberServiceImpl implements MemberService {
    private final MemberRepository memberRepository = new MemoryMemberRepository();
    
    @Override
    public void join(Member member) {
        memberRepository.save(member);
    }
    //...
}
```

- 위 코드에서 `memberRepository.save(member)` 호출 시 다형성에 의해 `MemoryMemberRepository`의 save 메서드가 실행됨
- 다형성의 실제 작동 방식을 명확히 체감함

### 역할과 구현을 분리한 설계의 장점

- 회원 저장소: 인터페이스(MemberRepository)와 구현체(MemoryMemberRepository)
- 할인 정책: 인터페이스(DiscountPolicy)와 구현체(FixDiscountPolicy)

요구사항의 변경 가능성이 큰 부분들(회원 데이터 저장소, 할인 정책)을 인터페이스로 분리함으로써:
- 구현체를 쉽게 교체 가능
- 클라이언트 코드 변경 없이 기능 확장 가능
- 기획이 확정되지 않은 상황에서도 개발 진행 가능

### 주요 도메인 설계
- **회원 도메인**: 회원 가입, 조회 기능
- **주문 도메인**: 상품 주문, 할인 적용 기능
- **할인 정책**: VIP 고정 할인 금액 적용

## 느낀점
객체 지향 프로그래밍에서 인터페이스와 구현체를 분리하는 설계가 얼마나 중요한지 실감했다. 기존에 개발할 때는 역할과 구현을 적절히 나누지 않았는데, 이번 강의를 통해 앞으로는 인터페이스와 구현 클래스를 잘 분리해서 개방-폐쇄 원칙(OCP)을 더 잘 지키는 코드를 작성해야겠다는 생각이 들었다.
